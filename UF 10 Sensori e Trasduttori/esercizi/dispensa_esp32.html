ADC 12 bit · Attenuazione programmabile · Media digitale · Calibrazione lineare

Questo modulo analizza in dettaglio la catena di misura completa per la temperatura con il sensore LM35, questa volta su piattaforma ESP32. Rispetto ad Arduino UNO esistono differenze sostanziali: l'ADC è a 12 bit invece di 10, la tensione di riferimento è 3.3 V invece di 5 V, l'ingresso analogico è configurabile con diversi livelli di attenuazione, e solo alcuni GPIO possono essere usati come ingressi ADC. Tutte queste differenze modificano significativamente i calcoli e le scelte progettuali.

1.2Il Sensore LM35

Il LM35 (Texas Instruments) genera una tensione d'uscita linearmente proporzionale alla temperatura in gradi Celsius: \(V_{\text{out}}(T) = S \cdot T\) con sensibilità \(S = 10\,\text{mV/°C}\).

Il LM35 alimentato a 3.3 V (pin 3V3 dell'ESP32) produce tensioni da 0 V (a 0 °C) a 1.50 V (a 150 °C). Questo range è completamente interno all'intervallo 0–3.3 V dell'ADC ESP32, quindi non servono resistori di partizione. Con attenuazione a 0 dB (range 0–1.1 V) il segnale copre fino a 110 °C senza saturare — scelta ottimale per la maggior parte delle applicazioni.

1.3L'ADC dell'ESP32 — Caratteristiche Dettagliate

L'ESP32 monta un ADC SAR (Successive Approximation Register) a 12 bit con architettura multi-canale multiplexata. La tensione di riferimento interna è nominalmente 3.3 V, ma il range effettivo in ingresso dipende dal livello di attenuazione selezionato.

La relazione di trasferimento ideale è:

$$D = \left\lfloor \frac{V_{\text{in}}}{V_{\text{ref}}} \cdot (2^{12} - 1) \right\rfloor = \left\lfloor \frac{V_{\text{in}}}{3.3\,\text{V}} \cdot 4095 \right\rfloor$$

e la relazione inversa per ricostruire la tensione:

$$V_{\text{in}} = \frac{D}{2^{12} - 1} \cdot V_{\text{ref}} = \frac{D}{4095} \cdot 3.3\,\text{V}$$

1.4Attenuazione Programmabile — Scelta del Range di Ingresso

Una caratteristica unica dell'ESP32, assente in Arduino UNO, è la possibilità di impostare l'attenuazione del preamplificatore in ingresso all'ADC tramite analogSetAttenuation(). Questo modifica il range di tensione accettabile sull'ingresso analogico:

ADC_6db

ADC_11db

Scelta ottimale per questo esercizio

Con il LM35 alimentato a 3.3 V, l'uscita massima è 1.50 V (a 150 °C). La scelta ottimale è ADC_2_5db (range 0–1.5 V): copre l'intero range del sensore con la massima risoluzione effettiva. Se si vuole misurare solo fino a 110 °C, si può usare ADC_0db (range 0–1.1 V) per avere ancora meno rumore. L'attenuazione va impostata prima della prima analogRead().

La scelta dell'attenuazione modifica direttamente la costante \(V_{\text{ref,eff}}\) nella formula di conversione, sostituendo 3.3 V con il limite superiore del range selezionato:

$$V_{\text{in}} = \frac{D}{4095} \cdot V_{\text{ref,eff}}$$

1.5Funzionamento dell'ADC SAR — Passo per Passo

Il convertitore SAR opera con un algoritmo di bisezione binaria iterativa: ad ogni ciclo di clock confronta la tensione campionata con il valore generato da un DAC interno, aggiornando un registro a scorrimento bit per bit, dal più significativo (MSB, bit 11) al meno significativo (LSB, bit 0).

Per un ADC a 12 bit il processo richiede 12 confronti successivi più overhead, per un totale tipico di 15–16 cicli di clock per conversione:

Ciclo 1 (MSB)
Il DAC produce \(V_D = V_{\text{ref}}/2 = 1.65\,\text{V}\). Se \(V_{\text{in}} > 1.65\,\text{V}\) → bit 11 = 1, altrimenti → bit 11 = 0.
½ del range
Ciclo 2
Il DAC aggiunge o sottrae \(V_{\text{ref}}/4 = 0.825\,\text{V}\). Il confronto determina il bit 10.
¼ del range
Ciclo k
Il DAC aggiusta di \(V_{\text{ref}}/2^k\). Il confronto determina il bit \((12-k)\).
\(1/2^k\) del range
Ciclo 12 (LSB)
Il DAC aggiusta di \(V_{\text{ref}}/4096 \approx 0.805\,\text{mV}\). Determina il bit 0.
risoluzione finale
Risultato
Il registro a 12 bit contiene il codice digitale finale, accessibile con \(\texttt{analogRead()}\).

1.7Esercizio 1 — Risoluzione e Errore di Quantizzazione (ESP32, 12 bit)

Risoluzione in tensione con attenuazione ADC_2_5db

Con l'attenuazione ADC_2_5db il range effettivo è \(V_{\text{ref,eff}} = 1.5\,\text{V}\). Il valore di 1 LSB è la variazione minima di tensione che provoca un cambiamento di un'unità nel codice digitale:

Definizione
$$\text{LSB} = \frac{V_{\text{ref,eff}}}{2^n - 1}$$
formula generale
Sostituzione
$$\text{LSB} = \frac{1.5\,\text{V}}{4095} = 3.663 \times 10^{-4}\,\text{V}$$
n=12, V_ref,eff=1.5 V
Risultato
$$\boxed{\text{LSB} \approx 0.366\,\text{mV}}$$

Risoluzione in temperatura

Invertendo la relazione del LM35 (\(V_{\text{out}} = S \cdot T\), con \(S = 10\,\text{mV/°C} = 0.01\,\text{V/°C}\)):

Relazione
$$\Delta T_{\min} = \frac{\text{LSB}}{S} = \frac{0.366\,\text{mV}}{10\,\text{mV/°C}}$$
Risultato
$$\boxed{\Delta T_{\min} = 0.0366\,\text{°C} \approx 0.037\,\text{°C per LSB}}$$
~13× più preciso di Arduino UNO
▶
Risoluzione in temperatura con ADC_2_5db: ΔT ≈ 0.037 °C / LSB — circa 13 volte migliore rispetto ad Arduino UNO (0.49 °C/LSB).

Errore massimo di quantizzazione

L'errore di quantizzazione è sempre ±½ LSB, indipendentemente dalla risoluzione:

In tensione
$$e_{q,\max} = \pm\frac{\text{LSB}}{2} = \pm\frac{0.366\,\text{mV}}{2} = \pm 0.183\,\text{mV}$$
In temperatura
$$e_{q,\max} = \pm\frac{0.037\,\text{°C}}{2} \approx \pm 0.018\,\text{°C}$$
▶
Errore massimo di quantizzazione: e_q = ±0.018 °C
Errore di quantizzazione — confronto visivo Arduino UNO vs ESP32

La scala a gradini più fine (ESP32, blu scuro) segue molto più da vicino la rampa analogica (arancione) rispetto all'ADC a 10 bit (grigio). L'errore \(e_q\) è visivamente quasi trascurabile con 12 bit.

1.8Esercizio 1 — Decodifica del Codice ADC 512 su ESP32

Il codice 512 sull'ESP32 (ADC 12 bit, range 0–4095) ha un significato fisico molto diverso rispetto allo stesso codice su Arduino UNO (ADC 10 bit, range 0–1023). Dobbiamo rifare il calcolo con i parametri corretti dell'ESP32.

Procedura: dato il codice \(D = 512\) da analogRead() su ESP32, si percorre la catena: \(D \xrightarrow{(1)} V_{\text{in}} \xrightarrow{(2)} T\). Il denominatore è ora 4095, non 1023.

Passo 1 — Dal codice digitale alla tensione (con ADC_2_5db)

Formula inversa
$$V_{\text{in}} = \frac{D}{2^{12} - 1} \cdot V_{\text{ref,eff}}$$
12 bit, V_ref,eff dipende dall'attenuazione
Sostituzione
$$V_{\text{in}} = \frac{512}{4095} \cdot 1.5\,\text{V}$$
D=512, ADC_2_5db → V_ref,eff=1.5 V
Quoziente
$$\frac{512}{4095} = 0.12503\ldots$$
512 è circa ⅛ del range 0–4095
Risultato
$$V_{\text{in}} = 0.12503 \times 1.5\,\text{V} = 0.1875\,\text{V} = 187.5\,\text{mV}$$

Passo 2 — Dalla tensione alla temperatura

Inversione LM35
$$T = \frac{V_{\text{in}}}{S} = \frac{187.5\,\text{mV}}{10\,\text{mV/°C}}$$
S = 10 mV/°C
Risultato
$$\boxed{T = 18.75\,\text{°C}}$$
temperatura ambiente realistica!
▶
Codice ADC 512 su ESP32 (ADC_2_5db) → V_in = 187.5 mV → T = 18.75 °C
Confronto con Arduino UNO — stesso codice, temperatura completamente diversa

Su Arduino UNO, il codice 512 corrispondeva a 2.502 V → 250.2 °C (fisicamente impossibile per il LM35). Sull'ESP32 con ADC_2_5db, lo stesso codice 512 corrisponde a soli 187.5 mV → 18.75 °C, una temperatura perfettamente normale e misurabile! Questo illustra come la scelta della piattaforma e dell'attenuazione cambi radicalmente l'interpretazione dei dati.

1.10Esercizio 3 — Sketch ESP32: Lettura Istantanea LM35

lm35_esp32_basic.ino
/**
 * lm35_esp32_basic.ino
 * ─────────────────────────────────────────────────────────────────────────
 * Lettura della temperatura LM35 su ESP32 — GPIO34 (ADC1_CH6, input-only).
 * Attenuazione ADC_2_5db → range 0–1.5 V → copre l'intero range LM35.
 *
 * Catena di conversione:
 *   analogRead() → D (0÷4095)
 *   V_in  = D / (2^12 - 1) * V_REF_EFF            [V]
 *   T     = V_in / SENSITIVITY                     [°C]
 *
 * Output CSV:  t[ms], D[raw], V[mV], T[degC]
 * ─────────────────────────────────────────────────────────────────────────
 */
#include <Arduino.h>

// ── Configurazione hardware ───────────────────────────────────────────────
static constexpr uint8_t   PIN_LM35         = 34;        // GPIO34 = ADC1_CH6, input-only
static constexpr uint8_t   ADC_BITS         = 12;        // risoluzione: 9, 10, 11 o 12
static constexpr uint16_t  ADC_MAX_CODE     = 4095U;     // 2^12 - 1

// ── Parametri dell'attenuazione ───────────────────────────────────────────
// ADC_0db  → range 0–1.1 V  → copertura LM35 fino a 110 °C
// ADC_2_5db → range 0–1.5 V  → copertura LM35 completa (0–150 °C) ← SCELTA OTTIMALE
// ADC_6db  → range 0–2.2 V  → ADC non sfruttato al 100%
// ADC_11db → range 0–3.9 V  → bassa precisione per LM35
static constexpr float     V_REF_EFF        = 1.5f;      // [V] — coerente con ADC_2_5db
static constexpr float     SENSITIVITY      = 0.010f;   // [V/°C]  10 mV/°C del LM35
static constexpr uint32_t  SAMPLE_PERIOD_MS = 1000UL;

void setup() {
    Serial.begin(115200);

    // Imposta risoluzione ADC a 12 bit (default ESP32, ma esplicitarlo è buona norma)
    analogReadResolution(ADC_BITS);

    // Imposta attenuazione: ADC_2_5db → range 0–1.5 V, ottimale per LM35 su 3.3 V
    analogSetPinAttenuation(PIN_LM35, ADC_2_5db);

    Serial.println(F("# t[ms],  D[raw],  V[mV],  T[degC]"));
}

void loop() {
    // Passo 1 — Lettura codice ADC grezzo (0–4095 con 12 bit)
    const uint16_t rawCode = analogRead(PIN_LM35);

    // Passo 2 — Conversione codice → tensione analogica [V]
    //   V_in = D / (2^n - 1) * V_ref_eff
    const float voltage = (static_cast<float>(rawCode) / ADC_MAX_CODE) * V_REF_EFF;

    // Passo 3 — Conversione tensione → temperatura [°C]
    //   T = V_in / S  (inversione di V_out = S * T)
    const float tempC = voltage / SENSITIVITY;

    // Output CSV
    Serial.print(millis());
    Serial.print(F(",  ")); Serial.print(rawCode);
    Serial.print(F(",  ")); Serial.print(voltage * 1000.0f, 2);  // [mV]
    Serial.print(F(",  ")); Serial.println(tempC, 2);             // [°C]

    delay(SAMPLE_PERIOD_MS);
}

1.11Esercizio 4 — Media Digitale e Riduzione del Rumore

Modello statistico della media campionaria

Ogni campione \(x_i\) si modella come \(x_i = \mu + \eta_i\), dove \(\mu\) è il segnale utile e \(\eta_i\) è il rumore con \(E[\eta_i]=0\) e varianza \(\sigma^2_x\). La media su \(N\) campioni:

$$\bar{x} = \frac{1}{N}\sum_{i=1}^{N} x_i \qquad \Rightarrow \qquad \sigma_{\bar{x}} = \frac{\sigma_x}{\sqrt{N}}$$

Per N = 50 campioni:

Riduzione σ
$$\sigma_{\bar{x}} = \frac{\sigma_x}{\sqrt{50}} \approx \frac{\sigma_x}{7.07}$$
fattore di riduzione ≈ 7×
In dB
$$\text{Attenuazione} = 20\log_{10}\!\left(\sqrt{50}\right) \approx 17\,\text{dB}$$
▶
Media su N = 50 campioni: rumore casuale ridotto di √50 ≈ 7.07× (≈ 17 dB)

lm35_esp32_averaged.ino
/**
 * lm35_esp32_averaged.ino
 * ─────────────────────────────────────────────────────────────────────────
 * Media su N_SAMPLES campioni + calibrazione lineare.
 * T_cal = CAL_A * T_raw + CAL_B
 * ─────────────────────────────────────────────────────────────────────────
 */
#include <Arduino.h>

static constexpr uint8_t   PIN_LM35         = 34;
static constexpr uint16_t  ADC_MAX_CODE     = 4095U;
static constexpr float     V_REF_EFF        = 1.5f;      // [V] — ADC_2_5db
static constexpr float     SENSITIVITY      = 0.010f;   // [V/°C]
static constexpr uint16_t  N_SAMPLES        = 50U;
static constexpr uint32_t  INTER_SAMPLE_US  = 500UL;    // [µs] pausa tra campioni

// Coefficienti di calibrazione: T_cal = CAL_A * T_raw + CAL_B
static constexpr float     CAL_A = 1.0178f;           // pendenza (adimensionale)
static constexpr float     CAL_B = 0.46f;            // offset residuo [°C]

static float acquireAverage() {
    uint32_t acc = 0;
    for (uint16_t i = 0; i < N_SAMPLES; ++i) {
        acc += analogRead(PIN_LM35);
        delayMicroseconds(INTER_SAMPLE_US);
    }
    return static_cast<float>(acc) / N_SAMPLES;
}

void setup() {
    Serial.begin(115200);
    analogReadResolution(12);
    analogSetPinAttenuation(PIN_LM35, ADC_2_5db);
    Serial.println(F("# t[ms],  T_raw[degC],  T_cal[degC]"));
}

void loop() {
    const float avgCode = acquireAverage();
    const float voltage = (avgCode / ADC_MAX_CODE) * V_REF_EFF;
    const float tempRaw = voltage / SENSITIVITY;
    const float tempCal = CAL_A * tempRaw + CAL_B;

    Serial.print(millis());
    Serial.print(F(",  "));  Serial.print(tempRaw, 2);
    Serial.print(F(",  "));  Serial.println(tempCal, 2);
    delay(1000);
}

1.12Esercizio 5 — Calibrazione Lineare per Due Punti

Il modello affine \(T_{\text{corr}} = a \cdot T_{\text{mis}} + b\). I coefficienti si determinano dai due punti sperimentali:

errore residuo = 0

Ponte di Wheatstone con Estensimetro
ADC 12 bit ESP32 · Analisi metrologica · Risoluzione effettiva e limiti pratici

Analizziamo la catena di misura per la deformazione meccanica mediante un estensimetro in configurazione a quarto di ponte, con lettura tramite ADC a 12 bit dell'ESP32. L'obiettivo centrale è quantificare perché un segnale di pochi millivolt non possa essere misurato direttamente senza amplificazione e calcolare la risoluzione metrologica del sistema.

2.1Il Ponte di Wheatstone

Il ponte di Wheatstone rileva variazioni relative di resistenza con alta sensibilità, eliminando i disturbi comuni (common-mode rejection). In configurazione a quarto di ponte, tre rami sono resistenze fisse \(R_1 = R_2 = R_3 = R_0\), il quarto è l'estensimetro \(R_g = R_0 + \Delta R\).

$$V_{\text{out}} = V_{\text{exc}} \left[\frac{R_g}{R_1 + R_g} - \frac{R_2}{R_3 + R_2}\right]$$

Per \(\Delta R \ll R_0\), lo sviluppo al primo ordine fornisce:

Forma esatta
$$V_{\text{out}} = V_{\text{exc}} \cdot \frac{\Delta R / R_0}{4 + 2(\Delta R / R_0)}$$
semplificazione con R₁=R₂=R₃=R₀
Linearizzata
$$V_{\text{out}} \approx \frac{V_{\text{exc}}}{4} \cdot \frac{\Delta R}{R_0}$$
errore ∝ (ΔR/R₀)² ≈ 0.0001% per ε=1000 µε

2.2L'Estensimetro — Modello e Parametri

Il Gauge Factor GF quantifica la sensibilità dell'estensimetro: \(\Delta R / R_0 = GF \cdot \varepsilon\)

Formula
$$V_{\text{out}} \approx \frac{V_{\text{exc}}}{4} \cdot \frac{\Delta R}{R_0} = \frac{3.3\,\text{V}}{4} \times 2 \times 10^{-3}$$
Calcolo

2.5Esercizio 3 — Conversione ADC con ESP32 (ADC_11db)

Per il ponte di Wheatstone si usa l'attenuazione ADC_11db (range 0–3.9 V) perché il segnale può essere amplificato con un INA e occorre coprire un range più ampio.

1 LSB
$$\text{LSB} = \frac{V_{\text{exc}}}{2^{12}-1} = \frac{3.3\,\text{V}}{4095} \approx 0.805\,\text{mV}$$
con ADC_11db, V_ref,eff ≈ 3.3 V
N_LSB
$$N_{\text{LSB}} = \frac{V_{\text{out}}}{\text{LSB}} = \frac{1.65\,\text{mV}}{0.805\,\text{mV}} \approx 2.05\,\text{LSB}$$
▶
Il segnale occupa ≈ 2 LSB → misura non affidabile senza amplificazione (INA necessario)
Misura inaffidabile senza INA

Con soli 2 LSB, l'errore di quantizzazione è ±½ LSB = ±25% del segnale utile. Il rumore intrinseco dell'ADC ESP32 (1–3 LSB RMS) è comparabile con il segnale → SNR < 0 dB. Serve un INA (es. INA128) con guadagno G ≥ 100× per portare il segnale a ≈ 165 mV (≈ 200 LSB).

2.6Esercizio 4 — Analisi Metrologica

Minima deformazione teoricamente rilevabile

V_out,min = 1 LSB
$$V_{\text{out,min}} = 0.805\,\text{mV}$$
1 passo minimo ADC
ΔR/R min
$$\left(\frac{\Delta R}{R_0}\right)_{\min} = \frac{4 \times 0.805\,\text{mV}}{3300\,\text{mV}} = 9.76 \times 10^{-4}$$
inversione formula ponte
ε min
$$\varepsilon_{\min} = \frac{9.76 \times 10^{-4}}{2} = 4.88 \times 10^{-4} \approx 488\,\mu\varepsilon$$
inversione rel. GF
▶
Minima deformazione teorica rilevabile: ε_min ≈ 488 µε ≈ 500 µε

Fattori che degradano la risoluzione reale

  • Rumore ADC ESP32 (1–3 LSB RMS): causato da rumore termico e switching noise del regolatore DC-DC integrato. La ENOB effettiva è ≈ 10–11 bit su 12 → risoluzione reale ≈ 4× peggiore della teorica.
  • Non-linearità integrale (INL ±2–4 LSB): deviazioni sistematiche dalla retta ideale, non eliminabili con la media.
  • Instabilità di V_exc: ogni variazione di 1% di V_exc produce un errore di 1% sulla misura di deformazione.
  • Rumore termico (Johnson–Nyquist): \(v_n = \sqrt{4k_BT R \Delta f}\) — per R=120 Ω, T=300 K, Δf=10 kHz: \(v_n \approx 4.5\,\text{nV}/\sqrt{\text{Hz}}\).
  • Drift termico del ponte: il TCR delle resistenze altera l'equilibrio anche senza deformazione (zero drift).
  • Interferenze EMI: pickup su connettori non schermati — critico con segnali sub-millivolt.
Soluzione industriale

Catena professionale: INA (G ≥ 100×, CMRR > 100 dB) → filtro anti-aliasing 4° ordine → ADC dedicato 24 bit (AD7124, ADS1256) → riferimento di tensione a basso drift (<3 ppm/°C). Risoluzione effettiva raggiungibile: ≈ 1 µε.

2.7Sketch ESP32 — Acquisizione e Decodifica Strain Gauge

wheatstone_esp32.ino
/**
 * wheatstone_esp32.ino
 * ─────────────────────────────────────────────────────────────────────────
 * Lettura estensimetro a quarto di ponte — ADC1 GPIO34, 12 bit, ADC_11db.
 *
 * Catena di misura inversa:
 *   D (ADC) → V_out [V] → ΔR/R₀ → ε [µε]
 *
 * NOTA PROGETTUALE: senza INA il segnale occupa ~2 LSB (SNR < 0 dB).
 * In un sistema reale: aggiungere INA128 (G=100) prima del pin ADC,
 * e usare ADC_0db (range 0–1.1 V) per massima precisione sul segnale amplificato.
 * ─────────────────────────────────────────────────────────────────────────
 */
#include <Arduino.h>

static constexpr uint8_t   PIN_BRIDGE       = 34;        // ADC1_CH6 — input-only
static constexpr float     V_EXC            = 3.3f;      // [V] eccitazione ponte
static constexpr uint16_t  ADC_MAX_CODE     = 4095U;     // 2^12 - 1
static constexpr float     V_REF_EFF        = 3.3f;      // [V] range ADC_11db ≈ 3.9 V, usiamo V_EXC
static constexpr float     GAUGE_FACTOR     = 2.0f;      // GF strain gauge metallico
static constexpr uint16_t  N_SAMPLES        = 128U;
static constexpr uint32_t  INTER_SAMPLE_US  = 200UL;

static float acquireAverage() {
    uint32_t acc = 0;
    for (uint16_t i = 0; i < N_SAMPLES; ++i) {
        acc += analogRead(PIN_BRIDGE);
        delayMicroseconds(INTER_SAMPLE_US);
    }
    return static_cast<float>(acc) / N_SAMPLES;
}

void setup() {
    Serial.begin(115200);
    analogReadResolution(12);
    analogSetPinAttenuation(PIN_BRIDGE, ADC_11db);   // range 0–3.9 V per il segnale del ponte
    Serial.println(F("# t[ms],  V_out[mV],  epsilon[microstrain]"));
}

void loop() {
    // 1. Acquisizione e conversione in tensione
    const float avgCode = acquireAverage();
    const float vOut    = (avgCode / ADC_MAX_CODE) * V_REF_EFF;   // [V]

    // 2. Inversione modello ponte (linearizzato, ΔR/R ≪ 1):
    //    V_out = (V_exc/4)·(ΔR/R)  →  ΔR/R = 4·V_out / V_exc
    const float dRoverR  = (4.0f * vOut) / V_EXC;

    // 3. Calcolo deformazione:
    //    ΔR/R = GF·ε  →  ε = (ΔR/R) / GF
    const float eps_ue   = (dRoverR / GAUGE_FACTOR) * 1.0e6f;   // [µε]

    Serial.print(millis());
    Serial.print(F(",  "));  Serial.print(vOut * 1000.0f, 3);   // [mV]
    Serial.print(F(",  "));  Serial.println(eps_ue, 1);           // [µε]
    delay(500);
}
